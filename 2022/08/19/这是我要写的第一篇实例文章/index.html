上面头部信息里面的内容供代码识别，文章里面是不会展示的，除非填写错误，tags是为了给文章附上标签属性，要不然从主页里面找不到跟测试相关的文章

大概就是这样，我们生成在本地看一下，没问题之后再部署到github

这里说几个命令和复合命令

hexo cl 清空缓存 建议每次都做

hexo g 生成静态文件

hexo s 生成本地预览

hexo d 部署

复合命令格式

生成并且预览 hexo cl && hexo g && hexo s

生成并且部署 hexo cl && hexo g && hexo d
